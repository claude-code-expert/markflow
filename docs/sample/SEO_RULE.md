# 프론트엔드 React 컴포넌트 SEO 및 접근성 가이드라인 (SEO Rule)

이 규칙은 React 컴포넌트(Next.js 포함)를 작성하거나 리팩토링할 때 검색 엔진 최적화(SEO)와 화면 판독기 접근성(a11y)을 극대화하기 위해 반드시 지켜야 하는 코딩 표준입니다. AI는 코드 생성 시 다음 사항을 최우선으로 적용해야 합니다.

## 1. 시맨틱 HTML (Semantic Tags)
- **비의미적 태그 최소화**: `<div>`나 `<span>`의 무분별한 사용을 피하고, 구조와 의미에 맞는 시맨틱 태그(`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`)를 우선적으로 사용합니다.
- **제목 계층 구조 준수**: 페이지 내에 단 하나의 `<h1>`을 두며, `<h2>`부터 `<h6>`까지 건너뛰는 일 없이 순차적인 계층 구조로 작성합니다.
- **목록 사용**: 반복되는 항목이나 링크의 나열은 항상 `<ul>`, `<ol>`, `<li>` 또는 `<dl>`, `<dt>`, `<dd>` 태그를 사용해야 합니다.

## 2. 네비게이션과 링크 (Links & Routing)
- **올바른 링크 요소 사용**: 페이지 이동이나 라우팅을 구현할 때는 클릭 가능한 `<div>`나 `<button>`을 사용하지 않고, 반드시 앵커 태그(`<a>`) 또는 프레임워크의 `<Link>` 컴포넌트를 사용합니다.
- **유효한 href 속성 보장**: 모든 링크는 검색 엔진 크롤러가 경로를 추적할 수 있도록 유효한 URL(상대 또는 절대 경로)을 `href` 속성에 가져야 합니다. `href="#"`나 `javascript:void(0)`의 사용을 엄격히 금지합니다.
- **외부 링크 처리**: 외부 사이트로 연결되는 링크에는 반드시 `target="_blank"`와 보안/SEO 목적의 `rel="noopener noreferrer"` 속성을 함께 명시합니다.

## 3. 이미지 및 미디어 최적화
- **의미론적 대체 텍스트(alt)**: 모든 `<img>` 태그와 이미지 컴포넌트에는 이미지를 잘 설명하는 `alt="..."` 속성을 필수적으로 추가합니다. 장식 목적의 이미지일 경우 화면 판독기가 무시하도록 빈 문자열 `alt=""`을 할당합니다.
- **레이아웃 시프트(CLS) 방지**: 브라우저가 이미지 공간을 미리 계산할 수 있도록 가능한 한 `width`와 `height` 속성을 명시합니다.
- **레이지 로딩**: 초기 뷰포트 바깥에 있는 이미지는 기본적으로 `loading="lazy"` 속성을 적용하거나 Next.js의 `<Image>`와 같은 최적화 컴포넌트를 활용합니다.

## 4. 메타 데이터와 Head 요소 제어
- 컴포넌트가 새로운 페이지 또는 독립적인 뷰를 구성하는 경우, 해당 페이지 컨텍스트에 맞는 동적 `<title>`과 `<meta name="description">`을 삽입하도록 설계합니다.
- React의 경우 `react-helmet-async`를 사용하고, Next.js의 경우 Metadata API 또는 `next/head` 컴포넌트를 사용하여 SEO 정보와 Open Graph(`og:title`, `og:image`, `twitter:card` 등) 태그를 설정합니다.

## 5. 접근성 (A11y) 고도화 (SEO의 연장선)
- **상호작용 요소의 구분**: 클릭 등 액션이 발생하는 UI 컨트롤은 반드시 `<button type="button">`을 사용하며, 키보드의 `Tab` 키로 접근(Focus) 가능해야 합니다.
- **ARIA 속성 활용**: 시각적 디자인 요소로 인해 네이티브 HTML만으로 의미 전달이 불가능한 햄버거 메뉴, 아코디언 컴포넌트, 모달창 등에는 `aria-expanded`, `aria-hidden`, `aria-label`, `aria-controls` 등의 ARIA 속성을 상황에 맞게 올바르게 적용합니다.
- **폼(Form) 라벨링**: 모든 `<input>`이나 `<textarea>`에는 짝지어진 `<label>`을 사용하거나 (시각적으로 숨겨진 경우라도) `aria-label`을 통해 해당 필드가 무엇을 하는지 명확히 알립니다.
