R2 Worker 배포 & 연결 가이드

MarkFlow Editor의 이미지 업로드용 Cloudflare Worker를 배포하고, 아래 연결하기 섹션에서 Worker URL을 바로 입력해 연동합니다. 전체 절차는 약 5분 소요됩니다.

1

R2 버킷 생성

Cloudflare 대시보드 → R2 Object Storage → Create bucket

  • 버킷 이름: markflow-images (또는 원하는 이름)
  • 생성 후 Settings → Public access → Allow Access 활성화
  • 공개 URL 확인 (예: https://pub-xxxxxxxx.r2.dev)
2

설정 편집 (wrangler.toml)

apps/worker/wrangler.toml을 열고 아래 값을 수정합니다. 

# account_id = "YOUR_ACCOUNT_ID"  # 주석 해제 후 입력

[[r2_buckets]]
bucket_name = "markflow-images"    # 위에서 생성한 버킷 이름

[vars]
PUBLIC_URL = "https://pub-xxxxxxxx.r2.dev"  # 버킷 공개 URL
Account ID 확인: Cloudflare 대시보드 → 아무 도메인 클릭 → 우측 하단 API 섹션에서 복사.
3

의존성 설치 & 배포

cd apps/worker
npm install
npm run deploy

배포가 끝나면 터미널에 Worker URL이 출력됩니다. 

https://markflow-r2-uploader.YOUR_SUBDOMAIN.workers.dev
처음 배포 시 Wrangler가 Cloudflare 로그인을 요구하면 npx wrangler login 실행 후 다시 시도하세요.
4

에디터에 Worker URL 연결

3단계에서 받은 URL을 아래 "연결하기" 섹션에 붙여넣고 테스트 업로드 → 저장 순으로 완료합니다.

저장된 값은 브라우저의 localStorage에 보관되며, 에디터에서 이미지를 붙여넣거나 드래그할 때 자동으로 사용됩니다.

Worker URL 연결하기

배포한 Cloudflare Worker의 URL을 입력하고 테스트 업로드로 실제 동작을 확인한 뒤 저장하세요.

저장된 URL:

API 레퍼런스

POST /upload
필드타입설명
file File (multipart) 이미지 파일 (png, jpg, gif, webp, svg)

성공 응답

{ "success": true, "url": "https://pub-xxx.r2.dev/images/1234-uuid.png" }

에러 응답

{ "success": false, "error": "File too large: 15.2MB. Max: 10MB" }
GET /  또는  GET /health

상태 확인용 엔드포인트.

로컬 테스트

cd apps/worker
npm run dev
# → http://localhost:8787 에서 실행
curl -F "file=@test.png" http://localhost:8787/upload

참고: 이 페이지는 브라우저 localStorage에 mf-cf-worker-url 키로 Worker URL을 저장합니다. 동일 도메인의 MarkFlow 앱이 이 값을 자동으로 읽어 이미지 업로드에 사용합니다.
