Product Overview

MarkFlow KMS

CommonMark 0.28 기반 풀스펙 Markdown 에디터 + 팀 지식 관리 SaaS.
문서는 카테고리(폴더) 단위로 구조화되고, 상호 링크로 유기적 지식 네트워크를 형성합니다.

Data Architecture

데이터 모델

PostgreSQL 기반 관계형 DB. 문서 변경 이력은 별도 버전 테이블로 관리.

핵심 엔티티
TABLE users { id uuid -- PK email text -- UNIQUE name text avatar_url text created_at timestamptz updated_at timestamptz } TABLE workspaces { id uuid -- PK name text slug text -- URL slug owner_id uuid -- FK → users theme_css text -- 커스텀 CSS created_at timestamptz } TABLE workspace_members { workspace_id uuid -- FK user_id uuid -- FK role enum -- owner|admin|editor|viewer joined_at timestamptz }
TABLE categories { id uuid -- PK workspace_id uuid -- FK parent_id uuid -- 자기참조(중첩 폴더) name text order_index int -- 정렬 순서 } TABLE documents { id uuid -- PK workspace_id uuid -- FK category_id uuid -- FK (nullable) author_id uuid -- FK → users title text content text -- Markdown raw prev_doc_id uuid -- 이전 페이지 FK next_doc_id uuid -- 다음 페이지 FK is_deleted bool -- soft delete created_at timestamptz updated_at timestamptz } TABLE document_relations { doc_id uuid -- FK related_id uuid -- FK (연관 문서) rel_type enum -- related|prev|next }
버전 & 검색
TABLE document_versions { id uuid doc_id uuid -- FK version_num int content text -- 해당 버전 전체 내용 author_id uuid created_at timestamptz } TABLE document_tags { doc_id uuid tag text } -- Full-Text Search Index CREATE INDEX doc_fts_idx ON documents USING gin( to_tsvector('english', title || ' ' || content) );
링크 프리뷰 캐시
TABLE link_previews { url text -- PK (해시) og_title text og_desc text og_image text og_site text preview_type enum -- website|video|internal cached_at timestamptz expires_at timestamptz } TABLE comments { id uuid doc_id uuid -- FK author_id uuid -- FK selection jsonb -- {from, to} 위치 content text resolved bool created_at timestamptz }
Scalability Design

확장성 설계

문서가 수천, 수만 건으로 늘어날 때를 대비한 구조적 설계 원칙.

문서 관리 확장성

📁 카테고리 구조
  • 중첩 폴더: parent_id 자기 참조 + Closure Table 패턴으로 깊은 계층 빠른 조회
  • Drag-Drop 순서: order_index Fractional Indexing 방식으로 재정렬 시 전체 업데이트 방지
  • 폴더별 문서 수 제한 없음, 페이지네이션으로 UI 렌더링 최적화
🔍 검색 확장성
  • 소규모: PostgreSQL FTS (pg_trgm + tsvector)
  • 중규모 이상: Elasticsearch/Meilisearch 분리 도입
  • 한국어 지원: 형태소 분석기 (nori tokenizer) 적용
  • 검색 결과: 제목 > 첫 단락 > 본문 순 가중치
⚡ 실시간 협업
  • Yjs CRDT 기반 — 충돌 없는 병합, 오프라인 편집 지원
  • WebSocket 서버 (y-websocket) 별도 마이크로서비스로 분리
  • 연결 인원 > 30명 문서: 서버사이드 awareness 최적화
💾 버전 관리
  • 전체 스냅샷 방식 (초기) → diff 압축 저장 (확장 시)
  • 최신 100버전 유지, 이후 월별 1개 스냅샷으로 자동 압축
  • 버전 간 diff 표시: Google Docs 스타일 변경 추적
