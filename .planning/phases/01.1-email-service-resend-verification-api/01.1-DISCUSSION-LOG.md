## 이메일 범위

| Option | Description | Selected |
|--------|-------------|----------|
| 3곳 모두 교체 (추천) | 가입 인증 + 비밀번호 재설정 + 초대 이메일 모두 Resend로 교체. 한 번에 정리하면 일관성 유지. | ✓ |
| 가입 인증만 먼저 | 가입 인증 이메일 + 재발송 API만 먼저 구현. 비밀번호 재설정/초대는 다음 페이즈에서. | |
| 인증 + 비밀번호 | 가입 인증 + 비밀번호 재설정만. 초대 이메일은 invitation-service 변경 필요해서 비용 큼. | |

**User's choice:** 3곳 모두 교체 (추천)
**Notes:** None

| Option | Description | Selected |
|--------|-------------|----------|
| 텍스트 전용 (추천) | HTML 없이 플레인 텍스트로 링크만 포함. MVP에 충분하고, 이메일 클라이언트 호환성 문제 없음. | |
| 기본 HTML | 간단한 HTML 템플릿 (inline CSS). 로고 + 버튼 + 링크 정도. 브랜드 이미지 없음. | ✓ |
| React Email | React 컴포넌트로 이메일 디자인. Resend와 완벽 호환. 다만 추가 의존성과 설정 필요. | |

**User's choice:** 기본 HTML
**Notes:** None
