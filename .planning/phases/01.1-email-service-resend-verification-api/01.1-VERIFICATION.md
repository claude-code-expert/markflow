#### 1. 통합 테스트 RESEND_API_KEY 미설정 환경 재실행

**Test:** CI 환경 또는 RESEND_API_KEY가 설정되지 않은 터미널에서 전체 테스트 스위트 실행:
```bash
pnpm --filter @markflow/api test
```
또는 명시적으로:
```bash
RESEND_API_KEY="" pnpm --filter @markflow/api test
```
**Expected:** 전체 테스트 스위트 통과 (유닛 10개 + 통합 5개 포함)

**Why human:** 로컬 `.env.local`에 실제 RESEND_API_KEY가 설정되어 있어 vitest가 이를 자동 로드합니다. 실제 Resend API를 호출하면 `markflow.app` 도메인이 미인증 상태이므로 403이 반환되고, 이로 인해 2개 통합 테스트(`should resend verification email for unverified user`, `should allow verifyEmail with the new token after resend`)가 실패합니다. 코드 로직 자체는 올바르며 (RESEND_API_KEY="" 시 5/5 통과), .env.local 없는 환경에서 검증이 필요합니다.

**근본 원인 설명:**
- D-09 결정: 통합 테스트에서는 항상 폴백 모드 사용 (환경변수 미설정)
- 그러나 vitest가 `.env.local`을 자동 로드하여 테스트 환경에도 RESEND_API_KEY가 주입됨
- register 등록 시 Resend API 호출 → 403 반환 → throw new Error → catch 블록에서 로그만 남김
- 그러나 이 에러가 발생하면서 app 내부 DB 커넥션과 테스트 DB 커넥션 간 타이밍 이슈가 발생하여 afterUser가 undefined로 조회됨
- 해결책: vitest.config.ts에서 테스트 환경에서 RESEND_API_KEY를 강제로 unset하거나, CI에서만 실행
